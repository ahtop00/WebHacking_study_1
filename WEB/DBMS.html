What`s DBMS

데이터 베이스의 본질은 Input의 Create/Update/Delete 와 Output의 Read이다.

데이터 베이스는 일반적인 파일에 비해 데이터의 표준화 기능을 제공하고 접근하기 쉽게 만든다. 또한 자동화 및 보안을 제공해준다.

MySQL - DBMS의 한 종류

관계형 데이터 베이스 제품군 중 하나인 MySQL는 주요 DBMS중 세 손가락 안에 꼽힌다.

데이터를 저장하는 표(table)를 데이터베이스(스키마)가 있고, 그 스키마를 저장하는 데이터 베이스 서버가 있다.

처음에 MySQL을 실행할땐 cmd에서 MySQL이 있는 파일로 위치를 옮긴 뒤에 명령어(mysql -uroot -p)를 통해 실행한다.

데이터베이스를 생성할 때는 (CREATE DATABASE 데이터베이스이름;)

반대로 데이터베이스를 삭제할 때는 (DROP DATABASE 데이터베이스이름;)

데이터베이스를 보고 싶을 때는 (SHOW DATABASES;) 테이블을 보고 싶을 때는 (SHOW TABLES;)

(USE 데이터베이스이름;)을 통해 그 스키마에 대한 명령을 쓸 수 있다.

[CREATE TABLE 테이블이름( ] 을 통해 테이블을 만들 수 있다.
ex) CREATE TABLE topic( id INT(11) NOT NULL AUTO_INCREMENT,
title VARCHAR(100) NOT NULL,
description TEXT NOT NULL,
created DATETIME NOT NULL,
author VARCHAR(30) NULL,
profile VARCHAR(100) NULL, PRIMARY KEY(id));

DSEC 테이블 이름; 을 통해 테이블의 요소를 알 수 있다.

테이블에 데이터를 추가하고 싶을 땐,
ex) INSERT INTO topic (title,description,created,author,profile) VALUES('MySQL', 'MySQL is ...', NOW(),'Hwahu','developer');

SELECT * FROM topic;을 통해 데이터를 볼 수 있다. 이 *대신 다른 요소를 쓰면 그 데이터만 볼 수 있다. 또한 여러가지 문법을 통해 내가 원하는 방식으로 데이터를 볼 수 있다.

테이블에 데이터를 업데이트하고 싶을 땐,
ex) UPDATE topic SET author='ziho' WHERE id=4;

테이블에 데이터를 없애고 싶을 땐,
ex) DELETE FROM topic WHERE id=5;
